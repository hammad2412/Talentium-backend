Core System Design

Controller Layer

Handles the application's business logic. Each controller method corresponds to a specific API action, uses asyncHandler to propagate errors, and interacts directly with Mongoose models.

Route Layer

Defines endpoint paths and HTTP methods. Acts as the entry gate, chaining middleware (auth в†’ authorize в†’ validate) before delegating to controllers.

Model Layer

Mongoose schemas model every domain entity. Handles DB-level validation, default values, and relationships through references and population.

Middleware Layer

A composable pipeline of cross-cutting concerns executed on every matched request вЂ” ensuring security, consistency, and validation before any controller logic runs.

Security Architecture

рџЄ– Helmet

Sets a comprehensive suite of secure HTTP response headers to protect against common web vulnerabilities.

рџљ¦ Rate Limiting

Restricts each IP to 100 requests per 10 minutes using express-rate-limit, preventing brute-force and DDoS vectors.

рџ§№ HPP Protection

Uses hpp (HTTP Parameter Pollution) middleware to sanitize query strings and prevent parameter override attacks.

рџ”‘ JWT Authentication

Stateless access token verification on every protected route. Validates signature, expiry, and confirms the user is still active in the database.

рџЊђ CORS Policy

Strict origin whitelist вЂ” only pre-approved client origins are permitted to make cross-origin requests, blocking unauthorized frontend access.

рџЋ­ Role-Based Access

Every protected route explicitly declares which roles (candidate / recruiter) may access it via the authorize middleware.

Middleware System

auth.middleware.js

Extracts and verifies the JWT access token from the request header. On success, attaches the authenticated user object to req.user for downstream use.

authorize.middleware.js

A higher-order middleware factory that accepts one or more allowed roles. Blocks the request with 403 Forbidden if the authenticated user's role is not in the permitted list.

validate.middleware.js

Accepts a Joi schema and validates req.body against it before the controller executes. Returns a descriptive 400 Bad Request on schema violation.

pagination.middleware.js

A reusable pagination utility injected into listing endpoints. Reads page and limit query params, computes MongoDB skip/limit values, and supports population of referenced documents.

error.middleware.js

Global Express error handler positioned at the end of the middleware stack. Catches all errors propagated via next(err), uses the custom ErrorResponse class to format consistent JSON error responses.
