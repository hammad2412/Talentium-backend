Project Overview

Talentium Backend powers a job portal platform that connects candidates and recruiters. The backend system handles the full lifecycle of job seeking and hiring through a clean, secure API.

👤 For Candidates

Authentication, profile management, education & experience tracking, skills, projects, preferences, job discovery, and application submission.

🏢 For Recruiters

Company management, job posting, applicant review, and application status management with role-based access control.

🏗️ Architecture Philosophy

Built with a modular MVC-inspired architecture ensuring scalability, maintainability, and security — with every layer having a single clear responsibility.

Layer Responsibilities

📦 controllers/

Business logic handlers. Receive validated requests, interact with models, and return responses.

🛣️ routes/

Express router definitions. Wire HTTP methods and paths to controller functions with middleware chains.

🗄️ models/

Mongoose schemas and models. Define the shape, validation, and relationships of all data entities.

🛡️ middlewares/

Reusable request pipeline stages — authentication, authorization, validation, pagination, and error handling.

✅ validations/

Joi schemas that enforce the structure, types, and constraints of incoming request bodies before they reach controllers.

🔧 utils/

Shared helper functions — async error wrapping, custom error classes, slug generation, and profile completion scoring.

⚙️

Core System Design

Controller Layer

Handles the application's business logic. Each controller method corresponds to a specific API action, uses asyncHandler to propagate errors, and interacts directly with Mongoose models.

Route Layer

Defines endpoint paths and HTTP methods. Acts as the entry gate, chaining middleware (auth → authorize → validate) before delegating to controllers.

Model Layer

Mongoose schemas model every domain entity. Handles DB-level validation, default values, and relationships through references and population.

Middleware Layer

A composable pipeline of cross-cutting concerns executed on every matched request — ensuring security, consistency, and validation before any controller logic runs.

Security Architecture

🪖 Helmet

Sets a comprehensive suite of secure HTTP response headers to protect against common web vulnerabilities.

🚦 Rate Limiting

Restricts each IP to 100 requests per 10 minutes using express-rate-limit, preventing brute-force and DDoS vectors.

🧹 HPP Protection

Uses hpp (HTTP Parameter Pollution) middleware to sanitize query strings and prevent parameter override attacks.

🔑 JWT Authentication

Stateless access token verification on every protected route. Validates signature, expiry, and confirms the user is still active in the database.

🌐 CORS Policy

Strict origin whitelist — only pre-approved client origins are permitted to make cross-origin requests, blocking unauthorized frontend access.

🎭 Role-Based Access

Every protected route explicitly declares which roles (candidate / recruiter) may access it via the authorize middleware.

🛡️

Middleware System

auth.middleware.js

Extracts and verifies the JWT access token from the request header. On success, attaches the authenticated user object to req.user for downstream use.

authorize.middleware.js

A higher-order middleware factory that accepts one or more allowed roles. Blocks the request with 403 Forbidden if the authenticated user's role is not in the permitted list.

validate.middleware.js

Accepts a Joi schema and validates req.body against it before the controller executes. Returns a descriptive 400 Bad Request on schema violation.

pagination.middleware.js

A reusable pagination utility injected into listing endpoints. Reads page and limit query params, computes MongoDB skip/limit values, and supports population of referenced documents.

error.middleware.js

Global Express error handler positioned at the end of the middleware stack. Catches all errors propagated via next(err), uses the custom ErrorResponse class to format consistent JSON error responses.

Authentication System

  • Candidate & Recruiter registration with role assignment
  • Secure login returning a signed JWT access token
  • Current session retrieval via protected route
  • Logout and token invalidation
  • Token refresh endpoint for seamless session renewal
Method Endpoint Description
POST /api/v1/auth/login Authenticate user & return JWT
POST /api/v1/auth/register/candidate Register a new candidate
POST /api/v1/auth/register/recruiter Register a new recruiter
POST /api/v1/auth/refresh Refresh access token
POST /api/v1/auth/logout Invalidate current session
GET /api/v1/auth/me Fetch authenticated user

Performance Optimizations

📑 Pagination

All listing endpoints use server-side pagination — preventing full collection scans and keeping response sizes lean.

🔗 Population

MongoDB document population joins related resources in a single query:

  • Job → Company
  • Application → Candidate
  • Application → CandidateProfile
🏥 Health Endpoint

GET /health returns server status, environment, timestamp, and service owner — ready for uptime monitors.
